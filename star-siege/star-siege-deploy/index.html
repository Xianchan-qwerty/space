About the Game

Star Siege is a 2D top-down arcade space shooter. Take control of a starfighter and hold off relentless alien waves across 10 escalating levels.

Survive, score big, and earn coins to upgrade your ship in the Black Market. Only the best commanders make it through the Ascendant wave.

  • 10 intense waves with layered bosses
  • Persistent ship upgrades
  • Challenging debuffs
  • Level and boss battle synth music
  • Retro CRT arcade aesthetic

Project Documentation

Introduction

Star Siege is a fully browser-based 2D arcade space shooter developed as a final project for the Game Development course. It demonstrates core game development concepts — game loops, collision detection, state management, and responsive UI — built from scratch without a game engine.

The goal is to pilot a spaceship, destroy alien enemy waves, and earn coins to purchase persistent upgrades in the Black Market, all while achieving the highest possible score.

Design

The game follows an authentic retro arcade aesthetic using a dark palette, neon accent glows, and CRT screen effects.

  • CRT scanline overlay and global fullscreen support
  • Persistent Black Market upgrade system
  • Synergistic power-ups including magnet, bomb, shield, cleanse, and weapon boosts
  • 10 levels with unique speed, spawn, enemy, and boss configurations
  • Layered boss health bars with dedicated boss battle music
  • Particle explosion effects and screen shake on hit
  • Mobile on-screen touch controls
Tools & Technologies
  • PHP — Main file structure & dynamic variables
  • JavaScript — Game logic, canvas rendering, input
  • HTML5 Canvas — Game object drawing
  • CSS3 — Animations, dark theme, responsive layout
  • Bootstrap 5 — Page layout and UI components
  • Web Audio API — Procedural sound effects, level music, and boss tracks
  • localStorage — Persistent high score, coins, upgrades, and fallback leaderboard
  • XAMPP — Local PHP server for development
Challenges
  • Game loop timing — Solved with delta-time accumulator for consistent spawn rates at any frame rate.
  • State persistence — Managed persistent upgrades (Coins, Speed, Fire Rate) alongside session-based scoring via localStorage.
  • Power-up synergy — Implemented stacked weapon boosts, screen-clearing bombs, cleanse, and magnet pull for power-ups only.
  • Layered bosses — Bosses use x-health bars so high-level fights feel longer without hiding progress.
  • Sounds without files — Used Web Audio API oscillators for shots, impacts, level music, and intense boss tracks.
Testing

All core features were manually tested across browsers and devices:

  • ✅ Persistent upgrades and coin economy
  • ✅ Fullscreen API integration and CRT rendering
  • Tested: Synergistic power-up stacking and magnet pickup behavior
  • ✅ Bullet-enemy and enemy-player collisions
  • Tested: 10-level progression and game over trigger
  • Tested: High score, local leaderboard fallback, and score submission flow
  • Tested: Layered boss health bars and level-specific boss music
  • ✅ Mobile touch buttons (hold behavior)
  • ✅ Responsive layout on desktop, tablet, and mobile

Conclusion

Star Siege successfully meets all project requirements: a playable game with mechanics, progression, scoring, audio, mobile support, and a polished UI — built entirely without a game engine.

Current feature set:

  • Multiple enemy types and layered boss fights
  • Persistent upgrades, coins, high score, and pilot name
  • Online leaderboard via PHP + MySQL with local fallback
  • Procedural arcade audio for levels, bosses, and combat effects
